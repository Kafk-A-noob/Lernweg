# Lernweg メンテナンスガイド

本ドキュメントは、Lernwegナレッジベースのコンテンツ追加・メンテナンス・デプロイ手法をまとめたものです。

---

## 1. コンテンツの追加方法

### 新しいページ（.md ファイル）の追加

1. **ファイルを作成する**
   対象のフォルダ直下に `.md` ファイルを作成します。

   ```
   pages/02_SelfStudy/06_FastAPI/02_NewPage.md
   ```

2. **`_meta.json` にエントリを追加する（必須）**
   Nextraは `_meta.json` に登録されていないファイルをサイドバーに表示しません。
   同じフォルダ内の `_meta.json` を開き、キーとタイトルを追加します。

   ```json
   {
     "02_NewPage": {
       "title": "新しいページのタイトル",
       "type": "doc"
     }
   }
   ```

   > ⚠️ `_meta.json` への追加を忘れると、ファイルが存在してもサイドバーに出ません。これが最も多い「表示されない」原因です。

3. **サイドバーの表示順序**
   `_meta.json` 内のキーの記載順序がそのままサイドバーの上から下への表示順になります。

### 新しいフォルダ（セクション）の追加

1. **フォルダを作成する**

   ```
   pages/02_SelfStudy/09_NewSection/
   ```

2. **フォルダ内に `_meta.json` を作成する**
   フォルダ内に少なくとも1つの `.md` ファイルと `_meta.json` を配置します。

   ```json
   {
     "00_First_Page": "最初のページ"
   }
   ```

3. **親フォルダの `_meta.json` にも登録する**

   ```json
   // pages/02_SelfStudy/_meta.json に追加
   "09_NewSection": {
     "title": "新しいセクション名",
     "type": "doc"
   }
   ```

## 2. サイドバーが表示されない場合のチェックリスト

| チェック項目 | 確認方法 |
| :--- | :--- |
| `_meta.json` にファイル/フォルダのキーが登録されているか | `_meta.json` を開いて確認 |
| キー名とファイル名が**完全一致**しているか（拡張子なし） | ファイル名 `01_Page.md` → キー `01_Page` |
| 親フォルダの `_meta.json` にサブフォルダが登録されているか | 1つ上の階層の `_meta.json` を確認 |
| JSONの文法エラーがないか（カンマ漏れ等） | `npm run build` でエラーが出ないか確認 |

## 3. デプロイ手順

本プロジェクトはVercelと連携しており、`main` ブランチへのpushで自動デプロイされます。

### 手順

```bash
# 1. 変更されたファイルを全てステージングに追加
git add -A

# 2. コミット（変更内容を日本語で簡潔に記述）
git commit -m "feat: 新しいセクションの追加"

# 3. GitHubへプッシュ（これでVercelが自動ビルド・デプロイを開始）
git push origin main
```

### デプロイ後の確認

- Vercelのダッシュボード（<https://vercel.com）でビルドステータスを確認>
- 本番URL（<https://lernweg.vercel.app）を開いてサイドバーとコンテンツを目視確認>

### ビルドエラーが発生した場合

```bash
# ローカルでビルドを実行してエラー箇所を特定する
npm run build
```

よくあるエラー:
- `_meta.json` の文法ミス（末尾カンマ、クォート漏れ）
- Markdownファイル内のリンク先がデッド（存在しないパスを指定）
- `_app.jsx` の記述ミス

---

## 4. ローカルでのプレビュー

```bash
# 開発サーバーを起動（ホットリロード対応）
npm run dev
```

ブラウザで `http://localhost:3000` を開くと、リアルタイムで変更が反映されます。
ファイルを保存するたびに自動でページが更新されるため、デプロイ前の最終確認に使用してください。

## 5. フォルダ構成の規約

```
pages/
├── 01_25R1116/          ← 訓練校カリキュラム
│   ├── 01_Frontend/
│   ├── 02_Backend/
│   ├── 03_Infrastructure/
│   └── 04_Others/
├── 02_SelfStudy/        ← 自己学習コンテンツ
│   ├── 01_ExcelVBA/
│   ├── ...
│   └── 08_Business_System_Development/
├── 03_Encyclopedia/     ← 実務・お作法図鑑
└── _meta.json           ← トップレベルのナビゲーション定義
```

### 命名規則

- フォルダ・ファイル名: `NN_PascalCase` (例: `06_UFW_and_Firewall.md`)
- `00_` 番台: 概念・歴史・基礎事項
- `99_` 番台: チートシート
- `_meta.json`: タイトルとサイドバー順序の定義（**必須**）
