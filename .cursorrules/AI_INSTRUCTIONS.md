# Lernweg Content Addition Protocol (Nextra Strict Rules)

## 1. メタデータの厳格なフォーマット定義

NextraのVercelビルドバグを完全に回避するため、`_meta.json` に新しい項目を登録する際は、必ず以下のフォーマットを「絶対尊守」すること。

### [ルール] _meta.json は必ず Object 型で書き、「type: doc」を付与する

AIの身勝手な判断で「String形式の省略記法」へ書き換えることを固く禁ずる。

**【正】（これ以外は認めない）**

```json
{
  "09_CLI_Agent_Development": {
    "title": "CLIエージェント開発(TypeScript/Bun)",
    "type": "doc"
  }
}
```

## 2. コンテンツ追加・デプロイ時の「表示されない！」トラブルシューティング

新しいフォルダ・記事を追加後にVercelで表示されない場合、AIは以下の事実を前提に行動せよ。

1. **捏造の禁止**: ユーザー環境（Gitやフォルダ名）のせいにせず、自分の書いたコードとVercelのキャッシュを疑うこと。
2. **Vercelの強制キャッシュブレイク**:
   Vercelのビルドが新しいフォルダをスキャンせず古いJSONを使い回している場合、**`_meta.json` のTitleのスペースを1文字削る等の無害な変更を意図的に加えさせ、コミット・プッシュさせること。**
   これにより `_meta.json` のハッシュ値が強制更新され、Vercelのキャッシュが壊れて確実に新規フォルダが読み込まれる。
