# 01. hostsファイルと名前解決（ローカルDNS）

Web開発において、ローカルサーバー（自分のPC）で開発中のサイトにブラウザからアクセスする際、単に `localhost` ではなく、本番と似たようなカスタムドメイン（例：`mysite.local`）を使いたい場合があります。

その際に操作するのが **`hosts`（ホスツ）ファイル** です。

## 1. hostsファイルとは？

インターネット上では、「 URL（例: `google.com`）」を「IPアドレス（例: `142.250.xxx.xxx`）」に変換する **DNSサーバー** という巨大な電話帳が動いています。これを「名前解決」と呼びます。

しかし、自分のPCの中にある `hosts` ファイルは、**全世界のDNSサーバーよりも優先して読まれるローカルな電話帳**です。

ここに「このURLにアクセスしたら、このIPアドレス（自分のPCなど）に行け」と強制的にルールを上書きすることができます。

## 2. Windowsにおける hostsファイル の位置

Windowsマシンの場合、以下のパスに隠されています。

```text
C:\Windows\System32\drivers\etc\hosts
```

※このファイルを編集するには、メモ帳やVSCodeを **管理者権限** で立ち上げて開く必要があります。

## 3. hostsファイルの実務的な書き方

中身はただのテキストファイルです。一番下に追記する形で設定します。設定は即座に反映されます。

```text
# IPアドレス        割り当てたいドメイン名
127.0.0.1       localhost
127.0.0.1       my-test-app.local
192.168.1.10    dev-server.company.internal
```

- **`127.0.0.1`**: ループバックアドレスといい、「自分自身のPC」を指す特殊なIPアドレスです。
- 上記のように書くことで、ブラウザで `http://my-test-app.local` と打ち込んだ瞬間に、DNSサーバーへは聞きに行かず、自分自身のPC上（WSL版のApacheやDockerなど）へ通信が飛ぶようになります。

## 4. 授業で扱われる理由

LAMP環境（Linux + Apache + MySQL + PHP）を作る際、Apacheで複数のサイトを同時起動する（バーチャルホスト）設定を行います。

その際、「URLによって表示するサイトを切り替える」テストを行うため、ブラウザから自由に名前（ドメイン）をつけてローカルのApacheにアクセスさせる仕掛けが必要になります。その下準備操作です。
