# 03. ユーザー・グループ管理とsudo

Linuxは「完全に隔離された複数人のユーザーが同時に使う」ことを前提（マルチユーザーシステム）とした強固なセキュリティ設計を持っています。

## 1. 最重要ファイル： /etc/passwd と /etc/group

Linuxの世界では、ユーザーのリストなどはすべて「単なるテキストファイル」として保存・管理されています。

### `cat /etc/passwd`

システムに存在する全てのユーザーの情報が書かれています。
**書式:** `ユーザー名:パスワード(xで隠蔽):UID:GID:コメント:ホームディレクトリ:ログインシェル`

### `cat /etc/group`

すべてのグループの情報が書かれています。
**書式:** `グループ名:パスワード(x):GID:所属しているユーザーのリスト（カンマ区切り）`

## 2. UIDの概念（Linuxの階級システム）

システムは人間を「名前」ではなく「背番号 (UID: User ID)」で識別します。

- **`0`**: `root` ユーザー。神です。何でもできます。（Windowsでいう完全なAdministrator）
- **`1 - 999`**: システムユーザー。Webサーバー(`www-data`など)やDBなど、バックグラウンドのプログラムが「人間ではないが権限を分離するために使う」幽霊アカウント群です。
- **`1000 -`**: 一般ユーザー。人間がログインして使うためのアカウントです。

## 3. rootになる (sudo su -)

一般ユーザーは重要な設定ファイルを編集できません。神である `root` に変身するか、または `root` の権限を一時的に借りる必要があります。

- `sudo <実行したいコマンド>`: 今回だけ `root` の権限を借りて実行する。
- `sudo su -`: 今から自分自身が完全に `root` ユーザーへと変身（スイッチ）する。（環境変数等のプロファイルも全てrootのものに切り替わります）

## 4. ユーザーの追加と「sudo」権限の付与

授業のログにあった `student` ユーザーの関連操作です。

### ユーザーの新規作成と削除

```bash
# student というユーザーを新しく作る（ホームディレクトリも作られます）
sudo adduser student

# パスワードを変更する
sudo passwd student

# studentユーザーを削除する（ホームディレクトリのデータごと完全に消す）
sudo deluser --remove-home student
```

### グループへの追加（usermod）

一般ユーザーである `student` は、初期状態では `sudo`（管理者権限を借りる権利）を持っていません。
Linux（Ubuntu）では、**「`sudo` という名前のグループに所属している人物だけが `sudo` コマンドを使える」**というルールがあります。

よって、`student` に管理者権限を与えるには以下のコマンドを打ちます。

```bash
# -aG (アペンド・グループ): student を sudoグループ に追加する
sudo usermod -aG sudo student

# studentが今どのグループに入っているかを確認する
groups student
```
