# 03. MySQL と MariaDB の深層

LAMP環境の「M」を担う、関係第一の倉庫（リレーショナルデータベース）の中心技術です。

## <a id="mysql"></a> 1. 歴史：なぜ2つの名前があるのか？

**MySQL**は1990年代に誕生した、オープンソースの王様となるRDBMSです。しかし2010年、MySQLを開発していたSun Microsystems社が、商用主義のOracle社に買収されました。

「このままではMySQLがオープンソースではなくなり、有料化や閉鎖的な開発になってしまう！」と危惧したMySQLのオリジナル開発者（マイケル・ウィデニウス）は、買収直前にMySQLのソースコードをコピー（フォーク）して独立し、自分の娘の名前(Maria)をとって新しく **MariaDB** を設立しました。

そのため、現在のLinux環境では、パッケージ名がMySQLとなっていても、実態は完全互換のMariaDBが動いているケースが非常に多いです（構文などはほぼ100%同じです）。

## 2. 概念と動作原理 (InnoDBエンジン)

現代のMySQL/MariaDBの心臓部は **InnoDB（イノデービー）** というストレージエンジンです。

### トランザクション (ACID特性)

InnoDBの最大の強みは「トランザクション」のサポートです。
例えば金融システムで「Aさんから1万円引く」「Bさんに1万円足す」という処理の途中でサーバーが落ちた場合、InnoDBは**途中まで行った処理をすべて取り消し（ロールバック）、誰も損をしていない状態に完全回帰**させます。
これをACID特性（原子性、一貫性など）と呼び、リレーショナルDBが2026年現在でもメインデータベースとして使われ続ける最大の理由です。

### 行レベルロック

古いデータベースでは、誰か1人がデータを修正している間「テーブル全体」がロックされ、他の人は待たされていました。InnoDBは「まさに今修正しているその1行だけ」をロック（行レベルロック）できるため、世界中からの同時アクセスに耐えられます。

## <a id="php"></a> 3. PHP / Node.js などの言語との連携

MySQL自体は、ネットワーク越しにSQL文（テキスト）を受け取って結果を返すだけのプログラム（デーモン）です。
ここに対してPHPやNode.jsが接続するためには「ドライバー」が必要です。

- **PDO (PHP Data Objects)**
  PHPからMySQLに繋ぐための現代の標準規格。直接SQLの文字列にユーザーの入力値を結合する（SQLインジェクションの罠）のを防ぐため、必ず `?` や `:name` などの**プレースホルダ（プリペアドステートメント）**を用いて安全にクエリを実行する仕組みを備えています。
