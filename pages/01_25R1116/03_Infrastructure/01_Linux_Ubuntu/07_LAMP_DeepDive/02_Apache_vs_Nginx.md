# 02. Apache と Nginx の対比・深掘り

LAMP環境における「A」の主役であるApacheと、現代の覇権であるNginx（エンジンエックス）の違いを深掘りします。

## <a id="apache"></a> 1. Apache HTTP Server の真理

### バーチャルホスト (VirtualHost)

Apacheの最強の武器は「1台のサーバーで複数のドメインを運用できる」バーチャルホスト機能です。

```apache
<VirtualHost *:80>
    ServerName www.my-test-app.local
    DocumentRoot /var/www/my-test-app/public
</VirtualHost>
```

### htaccess による分散設定

Apacheのもう一つの特徴は、各フォルダの中に `.htaccess` というファイルを置くことで、Webサーバーのメイン設定を「ディレクトリ単位」で上書きできることにあります。
- これにより、レンタルサーバー等で管理者権限を持たない一般ユーザーでも自由にURLのリダイレクトやBASIC認証を設定できます。
- ❌ **アンチパターン**: `.htaccess` は便利ですが、アクセスがあるたびにフォルダを遡って設定ファイルを読み込むため**非常に低速**になります。自社インフラであれば、メインの `.conf` に全て書き込むのがベストプラクティスです。

## <a id="nginx"></a> 2. 2026年覇権：Nginxの台頭

現代では、LAMPのAがNginxに置き換わった **LEMP環境** (L:Linux, E:Engine-x, M, P) が主流です。

### アーキテクチャの違い

- **Apache (マルチプロセス/スレッド駆動)**
  アクセスが来るたびに、スーパーのレジ係（プロセス）を新しく1人増やして対応します。1万人が同時に来るとレジ係が1万人になり、サーバーのメモリがパンクします（C10K問題）。
- **Nginx (イベント駆動 / 非同期アーキテクチャ)**
  少数の有能なレジ係が、注文（イベント）だけを次々と非同期で受け取り、裏に流し続けます。Node.jsと同じ非同期パラダイムであるため、大量の同時接続でもメモリを全く消費せず爆速で捌けます。

### リバースプロキシとしての役割

Nginxは、Next.jsやNode.js（Express）で作られたアプリの手前（前段）に配置し、「画像ファイルなどの静的資産はNginxが超速で返し、動的なAPIリクエストだけをNode.js等に流す」というキャッシング・中継器（リバースプロキシ）として利用されるのが2026年の絶対的な定石です。
