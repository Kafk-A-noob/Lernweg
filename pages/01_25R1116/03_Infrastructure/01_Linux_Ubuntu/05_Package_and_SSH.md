# 05. パッケージ管理(apt)とSSH

Linux(Ubuntu)をサーバーとして運用するための基本技術についてです。

## 1. パッケージ管理システム (apt)

WindowsではWebサイトから「.exe形式のインストーラー」をダウンロードしてダブルクリックしますが、現代のLinux（およびMacのHomebrew等）では、全てのアプリをコマンドラインから一括でダウンロード・インストール・アップデートするのが標準です。

Ubuntu等のDebian系Linuxで使われているのが `apt`（Advanced Package Tool）です。
※JS開発における `npm` と概念は完全に一致しています。

### 基本コマンド

```bash
# データベース(辞書)の最新化 (npm でいうレジストリ同期)
sudo apt update

# アプリのインストール (npm install 相当)
sudo apt install apache2 php mysql-server

# インストール済みの全アプリを最新版にアップデート
sudo apt upgrade
```
※システム全体にインストールを行うため、常に `sudo`（管理者権限）が必要です。

## 2. SSH (Secure Shell) と遠隔操作

サーバー機の入った「Linuxの実機」は、通常どこか冷房の効いたデータセンター（あるいはAWS・GCP等のクラウド上）に置かれています。モニターもキーボードも繋がっていません。

そのため、自分の手元のPC（Windows等）からサーバーへネットワーク越しに安全に入り込み、コマンドを打ち込む仕組みが **SSH** です。

### sudo apt install ssh
授業でこのコマンドを打ち込んだ理由は、自分の構築しているUbuntu仮想マシンに「外から別のPCがログインしてこれるようにするための『受け付けドア（SSHサーバー）』」をインストールした、ということです。

### SSHの接続の仕組み
SSHは通信の中身を全て暗号化（Secure化）するため、パスワードなどが途中で盗聴されても解読されません。

実務においては、パスワードすら使わず、「公開鍵・秘密鍵」という電子的な暗号キーを持ったPCからしかログインを許可しないという、より強固な設定（公開鍵認証）を行うのが絶対の鉄則となっています。
