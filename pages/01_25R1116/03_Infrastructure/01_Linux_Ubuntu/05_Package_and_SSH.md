# 05. パッケージ管理(apt)とSSH

Linux(Ubuntu)をサーバーとして運用するための基本技術についてです。

## 1. パッケージ管理システム (apt)

WindowsではWebサイトから「.exe形式のインストーラー」をダウンロードしてダブルクリックしますが、現代のLinux（およびMacのHomebrew等）では、全てのアプリをコマンドラインから一括でダウンロード・インストール・アップデートするのが標準です。

Ubuntu等のDebian系Linuxで使われているのが `apt`（Advanced Package Tool）です。
※JS開発における `npm` と概念は完全に一致しています。

### 基本コマンド

```bash
# データベース(辞書)の最新化 (npm でいうレジストリ同期)
sudo apt update

# アプリのインストール (npm install 相当)
sudo apt install apache2 php mysql-server

# インストール済みの全アプリを最新版にアップデート
sudo apt upgrade
```

※システム全体にインストールを行うため、常に `sudo`（管理者権限）が必要です。

## 2. SSH (Secure Shell) と遠隔操作

サーバー機の入った「Linuxの実機」は、通常どこか冷房の効いたデータセンター（あるいはAWS・GCP等のクラウド上）に置かれています。モニターもキーボードも繋がっていません。

そのため、自分の手元のPC（Windows等）からサーバーへネットワーク越しに安全に入り込み、コマンドを打ち込む仕組みが **SSH** です。

### sudo apt install ssh

授業でこのコマンドを打ち込んだ理由は、自分の構築しているUbuntu仮想マシンに「外から別のPCがログインしてこれるようにするための『受け付けドア（SSHサーバー）』」をインストールした、ということです。

### SSHの接続の仕組み

SSHは通信の中身を全て暗号化（Secure化）するため、パスワードなどが途中で盗聴されても解読されません。

実務においては、パスワードすら使わず、「公開鍵・秘密鍵」という電子的な暗号キーを持ったPCからしかログインを許可しないという、より強固な設定（公開鍵認証）を行うのが絶対の鉄則となっています。

---

## 3. 実務でのアンチパターンと失敗例

- ❌ **`apt update` をせずにいきなり `apt install` を実行する**
  - Linuxのパッケージリストのキャッシュが古いままインストールしようとして、「パッケージが見つかりません(404 Not Found)」といったエラーを出してハマるパターン。必ず直前に `apt update` を叩くのが鉄則です。
- ❌ **SSHのデフォルトポート(22番)を開けっぱなしにし、パスワード認証を許可したままネットに公開する**
  - AWSなどでサーバーを立て、パスワード認証をONにしたままにしておくと、数十分〜数時間以内に世界中のボットから「辞書攻撃（総当たり）」を受け、乗っ取られて仮想通貨のマイニングサーバー等として悪用されます（AWSから数十万円の請求が来ます）。

## 4. 2026年最新のベストプラクティス

1. **SSHの公開鍵認証と設定の厳格化**
   サーバー構築後、最初に行うべきセキュリティ設定です。`/etc/ssh/sshd_config` を編集し、`PasswordAuthentication no`（パスワード禁止）および `PermitRootLogin no`（rootでの直接ログイン禁止）を設定します。
2. **SSM等のマネージドな接続の活用（クラウド）**
   AWSの場合、もはやSSHのポート（22番）すら開けず、AWS Session Manager (SSM) などを利用してブラウザやAWS CLI経由でセキュアにログインする手法（踏み台サーバー不要のアーキテクチャ）がモダンインフラの主流になりつつあります。
3. **不要なパッケージは入れずにコンテナ(Docker)を配備する**
   現代では、ホストOS（Ubuntu本体）に `apt` で直接ApacheやPHPをゴリゴリ入れるのではなく、ホストOSにはDockerなどのコンテナエンジンだけを入れ、その上でコンテナとしてWebサーバーを動かすイミュータブル（不変）なインフラ設計がベストプラクティスです。

## 5. トラブルシューティング（よくあるエラー）

- **Q. `Could not get lock /var/lib/dpkg/lock-frontend` と出て apt が使えない**
  - **A.** バックグラウンドで自動アップデートが動いているか、過去の apt コマンドを強制終了させてしまってロックファイルが残っている状態です。数分待つか、OSを再起動してください。
- **Q. SSH接続時に `WARNING: UNPROTECTED PRIVATE KEY FILE!` という恐ろしいエラーが出る**
  - **A.** SSHに使おうとしている秘密鍵（`.pem` や `id_rsa`）の権限がガバガバ（誰でも読める状態）だと、SSHのソフトウェアが安全のために接続を拒否します。秘密鍵のパーミッションを `600` (所有者のみRead/Write可能) に変更してください。

## 6. 頻出メソッド・必須コマンド大全

システム管理・SSH接続で毎日使う必須コマンドです（暗記推奨）。

```bash
# 【apt】新しいアプリを入れる際の黄金コンボ（必ずupdateとセットで使う）
sudo apt update && sudo apt install -y <package_name>

# 【apt】インストール済みのパッケージから特定のものを検索する
apt list --installed | grep <名前>

# 【SSH】基本的なSSHログインの書式
ssh <user_name>@<ip_address>

# 【SSH】公開鍵（PEMファイル）を指定してAWSのEC2等にログインする
ssh -i /path/to/my-key.pem ubuntu@192.168.1.10

# 【SSH】自分のPCで新しく「秘密鍵と公開鍵のペア」を生成する（Ed25519方式が現在の最新標準）
ssh-keygen -t ed25519 -C "your_email@example.com"
```
