# 00. Pythonの原点と概念

## 1. 歴史と誕生背景

1991年、オランダのグイド・ヴァン・ロッサムによって開発されました。名前の由来はニシキヘビではなく、彼が好きだったイギリスのコメディ番組「空飛ぶモンティ・パイソン」です。
当時のプログラミング言語（C言語やC++等）は非常に難解で、書く人によってコードの形がバラバラになる問題がありました。
そこでPythonは、**「誰が書いても同じように読める美しいコード（The Zen of Python）」**を作ることを目的に設計されました。

長らく「教育用・趣味の言語」に近い立ち位置でしたが、2010年代以降の「データサイエンス」「機械学習（ディープラーニング）」の爆発的ブームにより、世界で最も人気のある言語の一つへと完全に化けました。

## 2. 概念と動作原理

### インタプリタとC言語のラッパー

Pythonは書いたコードをその場ですぐに実行する「インタプリタ言語」であり、実行速度は非常に遅いです。（C言語の数十倍遅い）。
しかし、AIやデータ分析の超重い計算においては、Pythonの裏側で動いている「C言語やC++で作られた激速ライブラリ（NumPyやPyTorch等）」を呼び出しているだけです。
つまり、Python自身は「C言語へのわかりやすい命令書（ラッパー）」として振る舞っているため、AI処理などの領域で遅さを感じさせない無敵の地位を築いています。

### インデント (字下げ) によるブロック表現

他の多くの言語が波括弧 `{}` で処理のカタマリ（if文の中身など）を表現するのに対し、Pythonは「タブやスペースの字下げ（インデント）」だけでブロックを表現します。これにより、物理的にコードを綺麗に書かないとエラーになるシステムが「誰が書いても美しいコード」を強制しています。

## 3. 強みと弱み (Pros/Cons)

- **強み (Pros)**: AI開発における「事実上の一強」です。豊富なライブラリにより、スクレイピング、自動化、バックエンド（Django/FastAPI）、機械学習まで何でも数行で書けます。初学者の最初の言語としても最適です。
- **弱み (Cons)**: スマホアプリの作成や、フロントエンド（ブラウザ上で動く処理）には絶望的に不向きです。また、実行速度が遅いため、高頻度取引システムのようなマイクロ秒を争うシビアなパフォーマンスが要求されるシステムには採用されません。

## 4. 本当の基礎事項

- **Global Interpreter Lock (GIL)**
  Pythonの歴史的な宿命とも言える仕様で、「同時に実行できるスレッドは常に1つだけ」という強固なロックがかかっています。そのため、複数コアのCPUを使っても純粋な計算処理（マルチスレッド）は速くなりません（マルチプロセス等で回避します）。
- **Pythonic（パイソニック）という思想**
  「シンプルな解決策は常に1つであるべき」という哲学があります。他言語のように裏技的な書き方で短くするよりも、Pythonらしい直感的な書き方（リスト内包表記など）をすることがプロの現場で強く推奨されます。
