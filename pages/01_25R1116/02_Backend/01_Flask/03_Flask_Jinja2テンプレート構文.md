# Jinja2 テンプレート制御構文 (実践)

本ドキュメントは、Flask標準のテンプレートエンジン「Jinja2」において、Python側の動的なデータ（リストや辞書、フラグ等）をHTML側で安全かつ柔軟に処理・描画するための各種制御構文についての学習総括である。(26'03-04学習分に基づく)

---

## 1. 制御構造 (Control Structures)

Jinja2では、HTMLの中にPythonライクな構文（`{% ... %}`）を埋め込むことで、変数値に依存した出し分けやループ処理を実現する。

### 条件分岐 (`if` / `elif` / `else`)

特定の条件（ログイン状態やデータの有無等）によって、画面に表示するHTMLタグ自体を切り替える。

```html
{% if is_logged_in %}
<p>ようこそ、{{ user_name }}さん！</p>
<a href="/logout">ログアウト</a>
{% else %}
<p>ゲストユーザーとして閲覧中</p>
<a href="/login">ログインはこちら</a>
{% endif %}
```

### ループ処理 (`for`)

Pythonから渡されたリストや辞書（DBから取得した複数件のレコード等）を反復処理し、同じ構造のHTML要素を量産する。

```html
<ul>
  {% for item in item_list %}
  <!-- loop.index (1始まりのカウント) などの便利な組み込み変数が使える -->
  <li>{{ loop.index }}: {{ item.name }} ({{ item.price }}円)</li>
  {% else %}
  <!-- リストが空だった場合のフォールバック（実務で多用） -->
  <li>商品がありません。</li>
  {% endfor %}
</ul>
```

## 2. 変数化とマクロ (`set`)

Jinja2テンプレートの内部で一時的な変数を作成し、複雑な判定ロジックの結果を格納したり、同じ計算式を複数回書く手間を省くことができる。

```html
<!-- アクティブなユーザーのみをフィルタリングした総数を変数に格納 -->
{% set active_count = users | selectattr('is_active') | list | length %} {% if
active_count > 0 %}
<p>現在 {{ active_count }} 人のユーザーがアクティブです。</p>
{% endif %}
```

【実務Tip】 複雑すぎるロジック（フィルタの連鎖や計算）をHTML(Jinja2)側に持たせることは推奨されない。極力、複雑なデータ加工はPython側(`app.py`)で完了させ、Jinja2側は「描画処理」に徹する（関心の分離）のが堅牢な設計である。

## 3. エスケープ処理とセキュリティ (`escape` / `safe`)

Webアプリケーションにおいて「ユーザーからの入力値」を画面に表示する際、最も警戒すべき脆弱性が**クロスサイトスクリプティング (XSS)** である。
Jinja2は、これに対する強力な防御レイヤーとして機能する。

### 自動エスケープ機能 (Autoescaping)

Jinja2はデフォルトで、変数（`{{ var }}`）として埋め込まれた値の中にあるHTMLの特殊文字（`<` や `>` など）を無害な文字（`&lt;` や `&gt;`）に自動変換する。
これにより、悪意のあるユーザーが `<script>alert('Hack!');</script>` と入力しても、ブラウザはそれをプログラムではなく「ただの文字列」として扱うため安全が担保される。

### 意図的なタグの適用 (`| safe`)

実務において「CMS（ブログシステム）の管理画面から入力されたリッチテキスト（文字色の変化や太字などのHTMLタグが含まれるデータ）」等、意図してHTMLタグを効かせたい場合は、`safe` フィルタを用いて自動エスケープを強制解除する。

```html
<!-- 危険：ユーザー入力値のタグが無害化されずに発動してしまう -->
<div>{{ user_input | safe }}</div>
```

【実務Tip】 `| safe` は文字通り「絶対の安全が保証されているデータ（管理者やシステム側が生成したデータ）」に対してのみ使用すること。一般ユーザーの入力値に対して `| safe` を付与することは、システムをXSSの脅威に直接晒す致命的な設定ミスとなる。

## 学習のまとめ

Jinja2の制御構文は、静的なHTMLを「アプリケーションとしての動的なビュー」に昇華させる根幹の技術である。
特にエスケープ処理の概念は、Webシステムにおけるセキュリティの第一歩であり、Next.js (React)の `dangerouslySetInnerHTML` 等、他のモダンフレームワークに移行しても共通して問われる不変の開発リテラシーである。
