# 00. Flaskの原点と概念

## 1. 歴史と誕生背景

2010年、Armin Ronacherによって開発されました。元々はエイプリルフールのジョーク企画として作られた非常に小さなフレームワークでしたが、そのシンプルさと使いやすさが大ウケし、事実上の標準ツールへと大出世しました。

当時のPython界隈には「Django（ジャンゴ）」という、「ORMから管理画面からユーザー認証まで全部盛り」の巨大で重厚なフレームワークが覇権を握っていました。
しかし、「APIをちょっと作りたいだけなのに、Djangoはルールが多すぎて学習コストが高すぎる」という開発者の声が高まっており、**「必要最小限（マイクロ）に徹し、必要な機能だけ後から足していく」**という思想を持つFlaskが待望されていたのです。

## 2. 概念と動作原理

### マイクロフレームワーク

Flask自身は非常に小さく、「URLを受け取り（ルーティング）」、「Pythonの関数を実行し」、「テキスト（HTMLやJSON）を返す」という基本骨格しか持っていません。データベース接続やユーザー認証の機能は最初から入っておらず、サードパーティ製の拡張プラグイン（SQLAlchemy等）を自分で組み合わせて作ります。

### WSGI (Web Server Gateway Interface) と Werkzeug

Flaskは `Werkzeug（ヴェルクツォイク）` というライブラリの上に作られています。
PythonのWebプログラムとWebサーバー（Nginx等）が会話するための標準規格「WSGI」に則って動いており、これによりどんなWebサーバーでもFlaskをセキュアに動かすことが可能になっています。

## 3. 強みと弱み (Pros/Cons)

- **強み (Pros)**: 学習コストが驚くほど低いです。たった5行のPythonコードだけでWebサーバーを立ち上げて「Hello World」を表示できます。AIからのデータを受け取る小さなAPI（マイクロサービス）を作る用途では右に出るものはいません。
- **弱み (Cons)**: ルールが少なすぎるため、大規模なエンタープライズシステムを作ろうとすると、チーム内で「ディレクトリ構成」や「DB設計」の書き方がバラバラになり、カオス化（無法地帯）しやすいのが最大の弱点です。また、近年は非同期処理（async/await）に特化した FastAPI というライバルにシェアを奪われつつあります。

## 4. 本当の基礎事項

- **デコレータ `@app.route` によるルーティング**
  Flaskの最大の特徴は、関数の直上に `@app.route('/hello')` と書くだけで、そのURLに来た時の処理を紐付けられることです。これを「デコレータ」と呼び、Pythonのメタプログラミングの強力さを示す代表例です。
- **Jinja2 テンプレートエンジン**
  Flaskが内蔵している、HTMLの中にPythonの変数を流し込むためのエンジンです。React全盛期の現在では「バックエンドでHTMLを生成する（SSR）」用途としてJinja2が使われる機会は減りましたが、メールの文面生成などでは現在も現役で使われます。
