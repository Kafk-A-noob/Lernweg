# TypeScript 基礎と前提知識 (00_TypeScript)

本ドキュメントは、現代のフロントエンド開発においてデファクトスタンダード (事実上の標準) となっている、JavaScriptに静的型付けの概念を付与した言語・TypeScriptの学習総括である。

---

## 1. なぜTypeScriptが必要なのか？

JavaScriptは変数に何を入れても許される (動的型付け) という特徴を持つが、これは数人で作る小規模なサイトでは便利でも、数十人で巨大なWebアプリケーション (Next.jsなど) を作る際には予期せぬ型エラー (数字が入るはずの場所に文字が入ってシステムが停止する等) を頻発させる最大の原因となる。

TypeScriptは、変数や関数の引数・戻り値にこれは数値専用これは特定の形をしたオブジェクト専用といった型 (Type) を事前に定義することで、**実行する前にエディタ (VSCode等) 上でエラーに気づける (静的型付け) **という強固な安全網を提供する。

---

## 2. 型定義の基本 (Type / Interface)

開発者が独自のデータ構造 (型) を定義するための仕組みとして、主に `type` と `interface` が存在する。

```typescript
// 商品データの形 (設計図) を型定義する
interface Product {
  id: number;
  name: string;
  price: number;
  isSale: boolean;
  description?: string; // ?をつけると、この項目は省略可能になる
}

// 定義したProduct型を使って変数を宣言する
const myItem: Product = {
  id: 1,
  name: "高級キーボード",
  price: 30000,
  isSale: false,
  // descriptionは省略可能なので書かなくてもエラーにならない
};

// 【エラー実例】
// myItem.price = "無料";
// ↑ number型を要求されている場所にstring(文字)を入れようとしたため、書いた瞬間にVSCodeが赤い波線で警告を出す
```

### 【実務Tip】 any型の乱用禁止

TypeScriptには、型チェックを完全に無効化する `any` という魔法の型が存在する。
しかし実務において `any` を使うことはTypeScriptを使っている意味をゼロにする行為として厳しく禁じられている (例外的なライブラリ連携時等を除く)。
型が複雑で分からないからといって `any` で逃げる癖をつけると、後から取り返しのつかないバグを生むため、必ず `type` や `interface` で構造を定義する習慣をつけること。

---

## 3. 関数における型の強力な恩恵

関数においても、引数と戻り値の型を明記することで絶大な効果を発揮する。

```typescript
// (x: 数字, y: 数字) を受け取り、必ず数字を返す関数
function addNumbers(x: number, y: number): number {
  return x + y;
}

// const result = addNumbers(10, "5"); // エラー: 2番目の引数はnumberである必要があります
```

このように、絶対に間違ったデータを受け付けない関数を構築できるため、他人が作った関数を呼び出す際にも何を渡せば何が返ってくるかが明確になり、チーム開発のスピードと品質が劇的に向上する。

## 学習のまとめ

JavaScriptのコードを書くたびに、いちいち型 (形) を定義しなければならないという初見の面倒くささは、システムが大きくなればなるほど型があるおかげで、エディタの入力補完 (予測変換) が完璧に効き、タイポ (打ち間違い) によるバグを100%防いでくれるという圧倒的なメリットに変わる。
ReactやNext.jsの開発現場においてはTypeScriptの採用率はほぼ100%に近いため、フロントエンドエンジニアにとって型概念の理解は必須科目である。
