# HTML_CSS 基礎と前提知識 (00_HTML_CSS)

本ドキュメントは、Web画面の構造 (HTML) と装飾 (CSS) を記述するためのマークアップ言語・スタイルシート言語について、訓練校での学習内容から抽出したフロントエンド開発の基礎統括である。

---

## 1. HTMLの役割と基本構造

HTML (HyperText Markup Language) は、ブラウザにここがタイトルここは段落ここは画像といった文書の構造を伝える役割を持つ。
最も基本的な骨格は以下のようになる。

```html
<!DOCTYPE html>
<html lang="ja">
  <head>
    <meta charset="UTF-8" />
    <title>Webサイトのタイトル</title>
    <!-- CSSファイルの読み込み -->
    <link rel="stylesheet" href="style.css" />
  </head>
  <body>
    <header>
      <h1>メインタイトル</h1>
    </header>
    <main>
      <p>ここに本文の段落が入ります。</p>
      <img src="flower.jpg" alt="花の写真" />
    </main>
  </body>
</html>
```

### セマンティックHTMLの重要性

`<div>` や `<span>` だけを多用するのではなく、`<header>`, `<main>`, `<nav>`, `<footer>` といった意味を持ったタグ (セマンティックタグ) を使うことが実務において強く推奨される。
これにより、検索エンジン(SEO)の順位向上や、音声読み上げソフト (アクセシビリティ) への適切な対応が可能となる。

### 【実務Tip】 エスケープ文字 (特殊文字)

画面上に `<` や `>` といった記号をそのまま表示したい場合、HTML内にそのまま書くとタグの始まりと誤認されてレイアウトが崩れてしまう。
実務現場では必ず以下のような特殊文字コード (エスケープ文字) を使用して安全に表示させる。

- `<` は `&lt;` (Less Than)
- `>` は `&gt;` (Greater Than)
- `&` は `&amp;`
- 半角スペースは `&nbsp;`

## 2. CSSの役割と記述ルール

CSS (Cascading Style Sheets) は、HTMLで作成された骨格に対して色、サイズ、配置などのデザインを適用する。

### 3つの適用方法と優先順位

1. **外部ファイル読み込み (推奨)**: `<link rel="stylesheet" href="...">`。全体でデザインを共有・管理できるため実務の基本。
2. **head内への直接記述 (非推奨)**: `<style>` タグ内に記述。そのページ内でのみ有効。
3. **インライン記述 (厳禁)**: HTMLタグに直接 `style="color: red;"` のように記述。

【実務Tip】 `style=` によるインライン記述は強制力が強すぎるため、後から外部CSSファイルでレスポンシブ対応 (スマホ画面用の変更) などをしようとした際に上書きできず、運用保守の最大の妨げ (技術的負債) となる。そのためプロの開発現場ではインライン記述は原則として禁止される。

### セレクタの基本

```css
/* 要素セレクタ: 全てのh1タグに適用 (影響が大きすぎるため実務ではあまり使わない) */
h1 {
  color: blue;
}

/* クラスセレクタ: class="alert-text" に適用 (実務での大黒柱) */
.alert-text {
  color: red;
  font-weight: bold;
}

/* IDセレクタ: id="main-header" に適用 (優先度が強すぎるため本当に1箇所だけの時に使う) */
#main-header {
  background-color: gray;
}
```

## 学習のまとめ

HTMLとCSSはプログラミング言語とは厳密には異なるが、すべてのWebアプリケーションの最も基礎となる視覚化 (出力) のインターフェースである。
後のReactやNext.jsなどの高度なフレームワークを用いた開発においても、最終的にブラウザが解釈するのはこのHTMLとCSSの束であるため、ここの構造を綺麗に書く (セマンティック・保守性) 意識がエンジニアリングの土台となる。
