# HTML_CSS Bootstrap実践 (01_Bootstrap)

本ドキュメントは、世界で最も広く利用されているCSSフレームワークの一つであるBootstrapの概念と、実務的なレイアウト構築におけるグリッドシステムの活用についての学習総括である。

---

## 1. CSSフレームワークとしてのBootstrap

Bootstrapは、よく使うプロレベルのデザイン (ボタン、アラート、カードなど) があらかじめ詰まったCSS・JavaScriptのパッケージ (道具箱) である。
エンジニアがゼロからCSSを書いて `border-radius` や `box-shadow` を調整しなくても、HTMLの `class` に決まった名前を指定するだけで、瞬時に洗練されたUIを構築できる。

### CDN (Content Delivery Network) による導入

手元にファイルをダウンロードせずとも、HTMLの `<head>` に専用のリンクタグを貼るだけで、世界中のサーバーから高速にBootstrapのファイルを読み込んで利用できる (CDN)。実務のモックアップ作成時によく使われる手法である。

---

## 2. 実用的なコンポーネント群

Bootstrapの恩恵を最も受けるのが、UIコンポーネントの呼び出しである。以下のようにクラスを付与するだけで機能する。

### ボタンとアラート

```html
<!-- 危険 (赤色) を示すボタン -->
<button class="btn btn-danger">削除する</button>

<!-- 情報 (青色) を示す通知エリア -->
<div class="alert alert-info">新着メッセージが1件あります。</div>
```

【実務Tip】 Bootstrapで使用される `primary`(メイン色/青), `secondary`(サブ色/灰), `success`(成功/緑), `danger`(危険/赤), `warning`(警告/黄) という色の意味づけによるクラス名規則は、現在世の中にあるほとんどのUIフレームワーク (Tailwind等) でも共通の思想として利用されているため、この概念を覚えておくことは非常に有益である。

## 3. レスポンシブとグリッドシステム (最重要)

Bootstrapを学ぶ最大の価値は、このグリッドシステムの理解にある。
これは画面の横幅を**12分割された透明なマス目 (カラム) **として捉え、スマホの時は幅を12マス全部使うPCの時は4マスずつ使って3列に並べるといったレスポンシブ・レイアウトを、CSSを一切書かずにクラス名だけで実現する仕組みである。

### グリッドの基本構造

グリッドシステムは必ず `container` 包み紙の中に `row` (行) を置き、その中に `col` (列) を配置するという厳密なルールで成り立つ。

```html
<div class="container">
  <!-- ① 全体を包む固定幅の箱 -->
  <div class="row">
    <!-- ② 1行の定義 (横幅12マスのレールを敷く) -->

    <div class="col-md-6">
      <!-- ③ 画面がmd(中程度)以上の時は 横幅6マス(半分)を使用 -->
      左半分のコンテンツ
    </div>
    <div class="col-md-6">
      <!-- ④ こちらも 横幅6マス(半分)を使用 -->
      右半分のコンテンツ
    </div>
  </div>
</div>
```

### 【実務Tip】 スマホ前提の設計 (Mobile First)

Bootstrapなど近年のフレームワークは、指定がない設定は、一番画面の小さいスマホ向けとして動作する。
例えば `<div class="col-md-4">` と書いた場合、スマホサイズでは12マス全幅 (縦に並ぶ) PCサイズ (md以上) になった時だけ4マス幅 (3列に並ぶ) という挙動になる。
このスマホでの見え方を先に決め、画面が広くなった時のルールを後から足す設計思想をモバイルファーストと呼び、現代のWeb開発の標準となっている。

また、`container-lg` 等を指定することで、超特大モニターで開かれた時も、コンテンツが間延びせず適度な中央幅に収まるといったプロフェッショナルな視認性の確保が瞬時に行える。
