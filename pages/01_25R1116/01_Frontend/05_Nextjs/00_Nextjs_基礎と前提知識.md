# Next.js 基礎と前提知識 (00_Next.js)

本ドキュメントは、Reactをベースにしつつ、実務のWebアプリケーション開発において必要な「表示速度」「SEO対策」「ルーティング設定」といった高度な機能をはじめから備えたフルスタック・フレームワーク「Next.js」についての学習総括である。

---

## 1. なぜReact単体ではなくNext.jsなのか？

React単体で作られたWebアプリ (SPA: Single Page Application) には、実務 (特に一般公開される商用サイト) において致命的となり得る大きな弱点が存在した。

**「初回アクセス時に、中身がからっぽのHTMLと巨大なJavaScriptが送られてきて、ブラウザ側で一生懸命画面を組み立てるため、最初の表示が遅い＆検索エンジンのロボット (SEO) が内容を読めない」** という問題である。

Next.jsは、この問題を「サーバー側で先にHTMLを作ってからブラウザに送る (SSR: サーバーサイドレンダリング) 」などの仕組みによって解決し、最高のユーザー体験とSEO最適化を両立させた「Reactの完全進化版」である。

---

## 2. App Router と ファイルベースルーティング

Next.js (バージョン13以降) の最大の特徴は、`app` ディレクトリを用いた非常に直感的な「ファイルベースルーティング」である。
React単体では別途「React Router」などの複雑なライブラリを導入する必要があったが、Next.jsでは「フォルダを掘って `page.tsx` を置くだけ」でURL (ページ) が完成する。

```text
src/app/
 ├── page.tsx          -> 「 / 」 (トップページ)
 ├── about/
 │    └── page.tsx     -> 「 /about 」 (アバウトページ)
 ├── products/
 │    └── [id]/
 │         └── page.tsx -> 「 /products/123 」 (動的ルーティング: 商品詳細など)
```

この「見たままがURLになる」という直感的な構造により、大規模なサイトでもページ管理が極めて容易になる。

## 3. Server Components (RSC) のパラダイムシフト

Next.js 13から導入された「React Server Components」は、これまでのフロントエンドの常識を覆した最強の機能である。

デフォルトで作成されるコンポーネントはすべて「サーバー側で実行される (ブラウザには結果のHTMLだけに行く) 」ようになる。
これにより、**「フロントエンドの構成要素 (コンポーネント) の中に、データベースに直接アクセスする処理を書いても安全」** という、かつての常識では考えられない開発が可能となった。

```tsx
// 【サーバーコンポーネント】 APIなどの裏側の処理を直接書ける！
export default async function ProductList() {
  // データベースから直接商品リストを取得 (通信内容などはブラウザから絶対に見えないので安全)
  const products = await db.product.findMany();

  return (
    <ul>
      {products.map((item) => (
        <li key={item.id}>{item.name}</li>
      ))}
    </ul>
  );
}
```

### 【実務Tip】 「use client」の使いどころ

ただし、サーバーコンポーネントはブラウザの機能 (`onClick` などのクリックイベントや、`useState` などの状態管理) を一切持てない。
そのため、実務においては「画面の骨格やデータ取得はサーバー部分 (デフォルト) で行い、ボタンをクリックして中身が動くような"インタラクティブな部品"のファイルの先頭にだけ `'use client'` と宣言してクライアントコンポーネント化し、2つをパズルのように組み合わせる」というアーキテクチャ設計力が強く求められる。

## 学習のまとめ

Next.js (特にApp Router) の登場により、フロントエンドエンジニアは「画面の見た目を作る人」から「バックエンドのデータ取得処理からインフラ (エッジレンダリング等) までをまるごと組み上げる、真のフルスタックエンジニア」としての役割を担うようになった。
現代のWebアプリケーション開発市場において最も強力かつ需要の高い技術であり、Reactの基礎 (StateとProps) を理解した上でこのNext.jsの仕様をマスターすることは、プロフェッショナルへの最速のチケットとなる。
