# JavaScript 基礎と前提知識 (00_JavaScript)

本ドキュメントは、Web画面上で「動き」や「裏側のデータ処理」を担当し、現代のすべてのモダンWebフレームワーク (React等) の土台となるプログラミング言語JavaScriptのコア機能についての学習総括である。

---

## 1. 変数宣言とデータ型

JavaScriptにおける値の保管箱 (変数) の定義方法は歴史的経緯により複数存在するが、使い分けのルールは現代において明確に決まっている。

### const と let (varの廃止)

```javascript
// 【実務Tip】 現代JSの絶対ルール: 基本はすべて const で宣言する
const userName = "Yamada"; // → 後から中身を書き換えられない (上書き不可)

// ループ処理や、どうしても後から中身を変える必要がある場合のみ let を使う
let score = 0;
score = 100; // → 上書き成功

// var は「スコープ (有効範囲) がガバガバで予期せぬバグの温床になる」ため、実務での使用は厳禁である。
```

### データ型と動的型付け

JSは「変数に何を入れても良い (文字でも数字でも配列でも) 」という動的型付け言語である。

- `Number` : 数字 (例: `25`)
- `String` : 文字列 (例: `"Hello"`)
- `Boolean`: 真偽値 ( `true` または `false` )
- `Array` : 配列。複数のデータを順列で管理 (例: `["Apple", "Banana"]`)
- `Object` : 辞書型。名前とデータのペア (例: `{ name: "Tanaka", age: 30 }`)

## 2. 厳密等価演算子 ( === )

条件分岐 (`if`) で「左と右が同じか」を比較する際、JS特有の罠が存在する。

```javascript
const ageStr = "20"; // 文字列の20
const ageNum = 20; // 数値の20

// ① 緩い比較 ( == ) : NG
if (ageStr == ageNum) {
  // 【危険】型が違っても、中身が一緒っぽければJSが勝手に変換して「同じ(true)」と判定してしまう
}

// ② 厳密比較 ( === ) : 実務での大鉄則
if (ageStr === ageNum) {
  // 型 (文字か数字か) まで完全に一致しないと「同じ」とは認識しない。こちらが false になる安全な判定。
}
```

【実務Tip】 意図しない型変換によるサイレントバグ (エラーを出さずに間違った計算を続ける最悪のバグ) を防ぐため、実務では `==` や `!=` は使用せず、必ず `===` や `!==` を使用する。

## 3. 制御構文：ループ処理 (繰り返し)

同じ処理を繰り返すためのループ処理には、状況に応じた使い分けがある。

### for と while

```javascript
// ① for 文: 「何回繰り返すか最初から決まっている」場合に使う
for (let i = 0; i < 5; i++) {
  console.log(i); // 0から4まで出力
}

// ② while 文: 「ある条件を満たす限り、回数無制限で繰り返す」場合 (ファイルの終端まで読み込む等) に使う
let hp = 10;
while (hp > 0) {
  hp -= 3; // hpが0以下になるまで攻撃処理がループする
}
```

### 配列専用の強力なループ (forEach / map / filter)

モダンなJSにおいては、配列の中身を1つずつ取り出す際は `for` 文よりも安全な配列専用メソッドを使うのが主流である。

```javascript
const fruits = ["Apple", "Orange", "Banana"];

// forEach: 順番に取り出して処理するだけ
fruits.forEach(function (fruit) {
  console.log(fruit);
});
```

## 4. 関数とデフォルト引数

処理のまとまり (関数) を定義する。変数として受け取る値を「引数 (ひきすう) 」と呼ぶ。

```javascript
// 単純な税抜き価格を計算する関数
function calcTax(price, taxRate = 0.1) {
  return price + price * taxRate;
}

// taxRate を省略した場合、自動的に「0.1 (10%)」がデフォルト引数として使われる
console.log(calcTax(1000)); // 1100 (10%で計算)
console.log(calcTax(1000, 0.08)); // 1080 (軽減税率8%で上書き計算)
```

【実務Tip】 実務において関数を設計する際、「引数が渡されなかった時に undefined になってエラーでシステムが停止する」事態を避けるため、上記のような「デフォルト引数 (初期値) 」を設定しておく手法が堅牢なプログラムの基本となる。

## 学習のまとめ

「基本は `const`」「比較は `===`」「`for`より配列メソッド」というルールは、単なる好みの問題ではなく「チーム開発において、他の人が読んでも予期せぬ事故が起きない安全なコードを書く」ための先人たちの知恵の結晶である。この堅牢性への意識が、JavaScriptを単なるWebのオモチャからエンタープライズ対応の言語へと進化させている。
