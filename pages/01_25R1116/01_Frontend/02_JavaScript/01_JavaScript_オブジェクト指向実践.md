# JavaScript オブジェクト指向実践 (01_OOP)

本ドキュメントは、大規模化するアプリケーション開発において、コードの再利用性と保守性を劇的に高める設計思想「オブジェクト指向プログラミング (OOP)」のJavaScriptにおける実装に関する学習総括である。

---

## 1. クラスとインスタンスの概念

「名前」「年齢」といったデータと、「喋る」「走る」といった処理 (メソッド) がバラバラの変数として存在すると、システムが大きくなった際に管理が破綻する。
これを「1つの塊 (オブジェクト) 」として整理・梱包するアプローチがオブジェクト指向である。

### クラス (設計図)

クラスとは「こういうデータと処理を持ったモノを作る」という「設計図・金型」である。

```javascript
// クラス名は大文字から始めるルール (PascalCase)
class User {
  // コンストラクタ (初期化処理)。newで量産される際に必ず一番最初に呼ばれる。
  constructor(name, age) {
    // this.〇〇 = は「これから生み出される自分自身のプロパティ (内部変数) 」を指す
    this.name = name;
    this.age = age;
  }

  // メソッド (そのオブジェクトが実行できる振る舞い)
  greet() {
    console.log(`私は${this.name}、${this.age}歳です。`);
  }
}
```

### インスタンス (実体)

クラスという設計図から、`new` キーワードを使って実際のデータを持った「実体 (インスタンス) 」を無限に量産できる。

```javascript
// 設計図から実体を2つ生み出す
const userA = new User("Yamada", 25);
const userB = new User("Suzuki", 30);

userA.greet(); // 「私はYamada、25歳です。」
userB.greet(); // 「私はSuzuki、30歳です。」
```

## 2. カプセル化による安全性確保 (ゲッターとセッター)

オブジェクトの中身の数値を、外部から誰でも勝手に書き換えられる状態 (`userA.age = -50;`など) は、バグやデータの矛盾を引き起こすため非常に危険である。
これを防ぐため、直接のアクセスを禁止し (カプセル化)、専用の「窓口」を通してのみ書き換えを許可する仕組みが必要となる。

### プライベートプロパティとアクセス制限

現代のJSでは変数名の頭に `#` を付けることで「クラスの外側からは絶対に見えないし、触れない魔法の結界」を張ることができる。

```javascript
class BankAccount {
  // クラス内部でのみ触れる厳重な金庫 (プライベートプロパティ)
  #balance;

  constructor(initialAmount) {
    this.#balance = initialAmount;
  }

  // 【窓口機能】 入金 (Setterの代わりとなるメソッド)
  deposit(amount) {
    if (amount <= 0) {
      console.error("エラー: マイナスの金額は入金できません");
      return;
    }
    this.#balance += amount; // 内部からなら触れる
  }

  // 【窓口機能】 残高照会 (Getterの代わりとなるメソッド)
  getBalance() {
    return this.#balance;
  }
}

const myAccount = new BankAccount(1000);
// myAccount.#balance = 10000000; // ← 外部からの不正な書き換えはエラーになりガードされる！
myAccount.deposit(500); // 正式な窓口を通せば処理される
```

【実務Tip】 「変数をむき出しにしない (プライベートにする) 」というカプセル化の考え方は、大規模なチーム開発において「他の人が不注意で重要な数値を壊してしまう事故」を未然に防ぐ、防弾チョッキのような役割を果たす。

## 3. 静的メンバー (Static)

通常、メソッドやプロパティは `new` して作られた「インスタンス (実体) 」に紐づくが、`static` キーワードを使うと「設計図 (クラス) そのもの」に紐づく機能を作ることができる。

```javascript
class MathConfig {
  // どのインスタンスでも変わらない絶対的な定数
  static PI = 3.14159;

  // インスタンスを作らなくても(newしなくても)直で使える便利ツール
  static calcCircleArea(radius) {
    return radius * radius * this.PI;
  }
}

// newしなくても、直接 クラス名.メソッド名 で呼び出せる (ユーティリティ関数として便利)
console.log(MathConfig.calcCircleArea(5));
```

## 学習のまとめ

JavaScriptにおけるクラス構文とオブジェクト指向は、長らく「関数」の組み合わせで強引に作られていた仕組みを、JavaやC#といった重厚な言語に近づける形で整理したものである。
このカプセル化やインスタンス化の思想を深く理解しておくことが、この後に続く「コンポーネント指向」を持つReactやNext.jsの高度な状態管理 (ステート) を読み解くための絶対的な前提条件となる。
