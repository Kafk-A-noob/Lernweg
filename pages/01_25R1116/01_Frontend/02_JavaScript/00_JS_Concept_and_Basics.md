# 00. JavaScriptの原点と概念

## 1. 歴史と誕生背景

1995年、Netscape Communicationsのブレンダン・アイクによってわずか10日間で開発されました。
本来の目的は「Webページにちょっとしたアニメーションや入力チェック（バリデーション）を持たせるためのおまけの言語」でした。Javaが流行っていたため、マーケティングの思惑で「JavaScript」という名前が付けられましたが、**Javaとは全く無関係の言語**です。

長らく「ブラウザでしか動かないオモチャの言語」と馬鹿にされてきましたが、2008年にGoogleが超高速なJSエンジン「V8」を発表したこと、そして2009年に「Node.js」が誕生してサーバー側でも動くようになったことで、世界で最も使われる覇権言語へと上り詰めました。

## 2. 概念と動作原理

### シングルスレッド・非同期駆動

JSの最も重要な概念は「**シングルスレッド（レジ係が常に1人しかいない）**」です。
そのため、API通信などで「結果が返ってくるまで待つ（ブロッキングする）」処理を書くと、画面全体がフリーズしてしまいます。
これを防ぐため、JSは「時間がかかる処理は裏のキッチン（Web API等）に投げ、自分は次の計算に進む」という**非同期処理（イベントループ）**の仕組みで動いています。

### ブラウザ内での実行 (DOM操作)

ブラウザにおけるJSの唯一にして最大の仕事は「**DOM (Document Object Model)** の書き換え」です。
HTMLを解析して作られたツリー構造（DOM）に対して、JSがアクセスし、要素の色を変えたり消したりすることで画面が動きます。

## 3. 強みと弱み (Pros/Cons)

- **強み (Pros)**: Webブラウザという「世界中のすべてのスマホとPCに入っているプラットフォーム」で唯一ネイティブに動く言語です。フロントエンド、バックエンド（Node.js）、スマホアプリ（React Native）、AI（AI SDK）まで一つの言語で全てを書ける「フルスタックの共通語」です。
- **弱み (Cons)**: 元が10日で作られた言語であるため、型の概念がなく「文字列のつもりだったのに数値が入って計算がおかしくなった ( `1 + "1" = "11"` ) 」といった実行時エラーが無限に発生します。これを解決するため、現在では**TypeScript**という型付きのスーパーセットを使うのが実務の絶対条件となっています。

## 4. 本当の基礎事項

- **`var` は使わない**: 過去の遺物です。スコープ（変数の有効範囲）がガバガバでバグの温床になります。現代の実務では `const`（再代入不可）を基本とし、どうしても中身が変わる場合のみ `let` を使います。
- **比較演算子の罠**: JSで一致を判定する際、`==` を使うと勝手に型を変換してしまいバグを生みます。（例： `0 == ""` がTrueになる）。必ず、型も値も厳密に比較する `===` を使用してください。
