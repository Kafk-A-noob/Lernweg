# ネイティブ言語 (Swift / Kotlin) 概要と歴史

スマートフォン向けアプリ開発において、「それぞれのOS (iOSとAndroid) の神様 (AppleとGoogle) が定めた公式の言語」を使って開発する手法を「ネイティブ開発」と呼ぶ。
モバイルアプリ開発業界を理解するための大前提となる知識をまとめる。

---

## 1. 2大ネイティブ言語の役割

### 🍎 iOS (iPhone / iPad) 陣営

- **公式リファレンス**: [Swift.org](https://www.swift.org/) / [Apple Developer Documentation](https://developer.apple.com/documentation/)
- **歴史**: かつては `Objective-C` (C言語に魔改造を施した非常に独特で難解な言語) が使われていた。
- **現在の標準**: **`Swift` (スウィフト)**
- **特徴**: Appleが「誰でも安全にモダンに書けるように」と2014年に発表した言語。型安全 (Type-Safe) であり、文法はTypeScriptやPythonの良いところを取り入れたような洗練されたデザインになっている。
- **開発環境**: `Xcode` (Mac専用の統合開発環境)。**Windowsパソコンでは絶対に開発できない**という致命的な制約がある。

### 🤖 Android 陣営

- **公式リファレンス**: [Kotlin Programming Language](https://kotlinlang.org/) / [Android Developers](https://developer.android.com/kotlin)
- **歴史**: かつては `Java` が標準言語として使われていた。
- **現在の標準**: **`Kotlin` (コトリン)**
- **特徴**: 冗長で記述が長くなりがちなJavaの欠点を補うためにJetBrains社 (IntelliJの開発元) が作り上げた、Javaと100%の互換性を持つモダンな言語。現在GoogleがAndroid開発における「第一言語 (Kotlin First) 」として公式認定している。
- **開発環境**: `Android Studio` (Windows / Mac 両方で動作)。

## 2. 両言語の共通点 (モダン言語の作法)

SwiftとKotlinは、生まれは違えど同世代のモダン言語であるため、文法や思想には驚くべき共通点がある。TypeScriptの良いところをさらに推し進めたような仕様となっている。

### Null安全 (Null Safety / Optional)

両言語の最大の特徴であり、プログラムが「突然落ちる (クラッシュする) 」原因の大半であるNull (値がない状態) を、コンパイル時に完全に防ぐ仕組みである。

- **`?` (オプショナル型)**: 変数の型の後ろに `?` をつけることで「この変数には値が入っていない (Nullの) 可能性があります」と宣言する。これを宣言しない限り、その変数には絶対にNullを入れることができない。

```kotlin
// Kotlinの例 (Swiftでもほぼ同じ)
var name: String = "Alice"
// name = null // ← コンパイルエラー！(安全)

var nickname: String? = "Al"
nickname = null // ← ? が付いているのでOK(危険を明示)
```

## 3. UI構築のパラダイムシフト (命令的 UI から 宣言的 UI へ)

ReactがWeb開発に革命を起こしたように、ネイティブ開発の世界でもここ数年でUI (画面) の作り方に劇的なパラダイムシフトが起きた。

### かつての作り方 (命令的 UI)

- **iOS**: `UIKit` (Storyboardという画面でマウスを使って部品を配置する)
- **Android**: `XML View` (XMLファイルでレイアウトを定義する)

### 現代の作り方 (宣言的 UI)

**「画面の見た目を、コード (関数) として記述し、状態が変わったら自動で再描画する」** という、完全にReactと同じ思想 (宣言的UI) に完全に移行しつつある。

- **iOSの最新標準**: **`SwiftUI`** (2019年〜)
- **Androidの最新標準**: **`Jetpack Compose`** (2021年〜)

つまり、**Reactの「状態 (State) に応じて画面を描画する」という概念を理解していれば、モダンなネイティブ開発のキャッチアップも非常に早い**という事である。

## 4. ネイティブ開発のメリットとデメリット

### ✨ メリット：圧倒的なパフォーマンスと最新機能

スマートフォンのOS (カメラ、GPS、Bluetooth、ARKit、最新のUIアニメーション等) の機能を100%引き出し、最も高速で滑らかに動作するのは、絶対にネイティブ言語である。
**「原神」のような高度な3Dゲームや、ミリ秒を争う金融系アプリ**などは、クロスプラットフォームではなくネイティブ技術 (またはUnity/C++等の専用プラットフォーム) で開発される。

### 💣 デメリット：コストと期間が「単純に2倍」かかる

最大の欠点はビジネス上のコストである。
「iPhone向け」と「Android向け」に、それぞれプロフェッショナルな Swift エンジニアと Kotlin エンジニアを雇い、**全く同じ機能のアプリを2つの言語・2つの環境で並行して書き、別々にテストをしなければならない**。
これが、予算が限られている受託開発において「ネイティブ離れ」が起きている最大の理由である。

## 5. なぜ「クロスプラットフォーム」が生まれたのか

「iOS用とAndroid用、どうせ画面の内容は同じなんだから、**1つのソースコードを書いたら両方で動くように翻訳 (または描画) してくれればいいじゃん！**」

という、世界中の経営者とエンジニアの悲願から生まれたのが、次の章で解説する `Flutter` や `React Native` などの**「クロスプラットフォーム (交差点) 開発環境」**である。
