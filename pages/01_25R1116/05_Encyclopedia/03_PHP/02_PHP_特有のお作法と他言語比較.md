# PHP 特有のお作法と他言語比較

Webサーバーへの埋め込み言語として発展してきた歴史を持つPHPは、JavaScriptやPythonを触ってきたエンジニアが「えっ？！」と直感に反して驚く独自の罠や文化 (お作法) を数多く持っている。本稿では他言語からの移行時に絶対知っておくべき概念を整理する。

---

## 1. 変数の `$` (ダラー記号) と文字列連結の罠

### 変数には必ず `$` をつける

- **公式リファレンス**: [変数 - PHP Manual](https://www.php.net/manual/ja/language.variables.basics.php)
  JSやPythonに慣れていると、宣言時や呼び出し時に `$` をつけ忘れて `SyntaxError` や `Warning` を出すのは誰もが通る道である。

```php
// JS: const name = "Alice";
// PHPの場合
$name = "Alice";
echo $name;
```

### 文字列の連結は `+` ではなく `.` (ドット)

- **公式リファレンス**: [文字列演算子 - PHP Manual](https://www.php.net/manual/ja/language.operators.string.php)
  JSやPythonでは `"Hello " + "World"` のようにプラス記号で文字を繋ぐが、PHPにおいて `+` は「算術加算 (純粋な足し算) 」専用の演算子である。

- **罠**: `echo "Hello " + "World";` と書くと、PHPは文字の中身を無理やり数値に変換して「0 + 0 = 0」と計算し、結果として `0` が出力されてしまう (PHP8以降は Fatal Error になる場合もある)。
- **お作法**: 文字列の連結には必ず `.` を用いるか、ダブルクォーテーションの変数展開機能を利用する。

```php
$first = "Hello";
$second = "World";

// 1. ドット演算子による連結
echo $first . " " . $second;

// 2. 波括弧を用いた展開 (推奨・JSのテンプレートリテラル ${} に近い)
echo "{$first} {$second}";
```

## 2. 【最重要】 配列 (Array) と連想配列 (Associative Array) の同一視

これは他のプログラミング言語 (JSやPythonを含むほぼすべて) と決定的に異なる、**PHP最大の独自仕様**である。

- **公式リファレンス**: [配列 - PHP Manual](https://www.php.net/manual/ja/language.types.array.php)

| 言語       | 順番を持つリスト        | キーつきの辞書                  |
| :--------- | :---------------------- | :------------------------------ |
| **JS**     | 配列 (`Array` `[1,2]`)  | オブジェクト (`Object` `{a:1}`) |
| **Python** | リスト (`list` `[1,2]`) | 辞書 (`dict` `{'a':1}`)         |
| **PHP**    | **配列 (`array`)**      | **配列 (`array`)**              |

PHPの世界では、0から順番に割り振られたただの配列も、文字列でキー管理された連想配列 (辞書) も、**全く同じ `array` 型 (内部的には順番付きマップ) ** として扱われる。

### どう違うのか？ 何が罠なのか？

JSではオブジェクト ( `{}` ) を扱いたいときと、配列 ( `[]` ) を扱いたいときで明確に構文やメソッド (`map` は配列専用) が分かれている。
しかしPHPではどちらも `配列` であるため、JSON等でデータを外部から受け取ったり出力したりする際に「これはリストなのか？それとも辞書なのか？」の判別が曖昧になり、バグの温床となる。

```php
// 普通の配列
$list = ["a", "b", "c"];

// 連想配列 (JSのオブジェクトに相当)
// => (アロー) 記号を用いてキーと値を紐付ける
$dict = [
    "name" => "Alice",
    "age" => 20
];
```

- **お作法**: PHPの連想配列データをAPIレスポンスとして `json_encode()` すると、JS上でObject `{}` に変換される。このデータ構造の橋渡し (マップとリストの違い) を常に意識して開発しなければならない。

## 3. HTMLへの埋め込み (テンプレートエンジン)

- **公式リファレンス**: [PHPのタグ - PHP Manual](https://www.php.net/manual/ja/language.basic-syntax.phptags.php)

React/Next.jsでは `JSX` を使い、Flaskでは `Jinja2` を使ってHTMLにデータを動的に埋め込んできたが、**PHP自体が巨大なテンプレートエンジン**である。

そのため、HTMLファイル (拡張子は`.php`) の途中に唐突に `<?php ... ?>` というタグを記述し、その中で変数を出力することが可能。 (※WordPressテーマ開発の基本思想となる)

```html
<!-- index.php -->
<ul>
  <?php foreach ($users as $user): ?>
  <!-- = (イコール) をつけることで、echo と同じ出力になる -->
  <li><?= htmlspecialchars($user["name"], ENT_QUOTES, 'UTF-8') ?></li>
  <?php endforeach; ?>
</ul>
```

- **罠**: JS等で用意されているXSS (クロスサイトスクリプティング) 対策の自動エスケープが、素のPHPには備わっていない。
- **お作法**: 入力値やDBから取得した文字列をHTMLとして出力 (`echo` や `<?=`) する際は、**必ず `htmlspecialchars()` 関数を介してサニタイズ (無害化) すること**。これを怠ったシステムは100%ハッキングの標的となる。
