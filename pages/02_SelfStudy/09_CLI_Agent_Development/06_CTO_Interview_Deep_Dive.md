# 最終面接対策：テクノロジーの深掘り（CTO視点）

このページでは、本課題で実装した技術を「なぜそれが動くのか」「なぜ他の技術では駄目なのか」というコンピューターサイエンスの根本的な理解へと昇華させるための深掘りを行う。

## 1. Zod はなぜ TypeScript の限界を超えられるのか？

### TypeScript（静的型付け）の罠

開発中はエディタ上で型エラーを警告してくれる TypeScript だが、JavaScript にトランスパイル（変換）された段階で **「型情報はすべて消滅（Type Erasure）」** する。
つまり、実行中の Node.js や Bun にとって、変数はただの `any` である。もし Gemini API や ユーザーが想定外の文字列（「ぎっとはぶ」等）を渡してきたとき、TypeScript はそれを防ぐ手段を持たない。

### Zod（実行時バリデーション）のメカニズム

Zod はソースコード上（静的）ではなく、**プログラム実行時（動的）のメモリ空間** でデータの中身を文字単位で検査する「パーサー」である。
`transform` パイプラインは単なる型の宣言ではなく、実行される「関数」として機能する。Zodの関所を通ったという「実行結果」をもって初めて、後ろに続くプロセスは「このデータは絶対に安全な `ServiceEnum` である」と確信できる。静的型と動的型の架け橋として Zod が極めて重要になる理由がここにある。

## 2. Function Calling と ReAct パターンの原理

### 従来のチャットボットとの違い

従来のLLMは「プロンプトを受け取り、テキストを返す」単方向の変換器に過ぎない。しかし、Function Calling を搭載したLLMは **「状態遷移機械（有限オートマトン）」** として動作する。

### ReAct (Reason + Act) の裏側

`agent.ts` における `while` ループは、以下のような自律駆動のサイクルを生み出している。

1. **Reason（推論）:** LLM が「Discordの状況を知るには `fetchServiceStatus` ツールを使うべきだ」と推論し、実行要求（JSON）を投げる。
2. **Act（行動）:** メインプログラムが LLM の代わりに API を叩く。
3. **Observe（観察）:** 取得したデータを LLM に差し戻す。
4. **再評価:** LLM は結果を見て「もう充分か（ループを抜ける）、まだ調査が必要か」を判断する。

LLM を単なる「テキスト生成器」ではなく、「自律的に判断して行動関数を呼び出す脳」として扱うのが、このアーキテクチャの核心である。

## 3. なぜ「マスキング層」の強制通過が必要なのか？

### プロンプトインジェクションとハルシネーション

LLM は本質的に言葉の意味（セマンティクス）を理解していない。次に来る確率の高い単語を並べているだけである。そのため、「内部APIが返したIPアドレスやAPIキー」がコンテキストに混入すると、意図せず回答テキストの中に出力してしまうリスク（ハルシネーションによる漏洩）が常に存在する。

### 「防御的プログラミング」としての破壊的サニタイズ

LLM の出力制御（「IPを隠して」とプロンプトで指示すること）は確率的であり、100%の保証はない。
対して、`masking.ts` による正規表現の置換は **「決定論的（Deterministic）」** である。LLM に文字列を渡す手前のネットワーク境界で、プログラム・アルゴリズム的に機密情報を `[MASKED]` に破壊してしまう。AI を信じない設計（ゼロトラスト的な考え方）こそが、セキュアなシステム構築の基礎となる。

## 4. Bun ランタイムの「コンパイル」とは何か？

### V8 エンジンの呪縛

Node.js を実行するには、OS に V8（Googleが作ったJavaScriptエンジン）がインストールされている必要がある。作ったCLIを他人に配布する場合、「まずNode.jsをインストールしてください」と言う必要があり、環境構築（Zero-setupの対極）が発生する。

### 実行環境のカプセル化

Bun の `bun build --compile` は、ユーザーが書いた TypeScript/JavaScript コードに加えて、**「JavaScriptCore（Appleが作った軽量エンジン）」と「ランタイム（Bun本体）」をC言語レベルの実行可能バイナリ（拡張子 .exe など）として１つのファイルにパッケージング（静的リンク）** してしまう。
これにより、相手のPCに何もインストールされていなくても、バイナリをダブルクリックするだけでエンジンが起動し、即座にツールが実行される。環境依存を破壊する強力なデプロイ手法である。
