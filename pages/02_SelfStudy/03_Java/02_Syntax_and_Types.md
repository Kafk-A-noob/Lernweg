# 02. 静的型付けと基本構文

Javaのコードは、常に「クラス」の中に書かれます。そして変数には厳格な「型」が存在します。

## 1. プリミティブ型と参照型

Javaにおける型は、大きく分けて2種類に分類されます。

### プリミティブ型（基本データ型）

値そのものを直接メモリに保持する、非常に軽量な型です。
- `int`: 整数（JSのNumberとは異なり、小数点は保持できません）
- `double`: 浮動小数点数（JSのNumberに近い）
- `boolean`: 真偽値 (`true` または `false`)
- `char`: 単一の文字（`'A'` など、シングルクォートで囲む）

### 参照型

オブジェクトの実体への「メモリの場所（参照先）」を指し示す型です。
- `String`: 文字列（`"Hello"` など、ダブルクォート。実はプリミティブではなくクラスです）
- 配列型 (`int[]` など)
- 自分で作成したクラス (`User` クラスなど)

```java
// プリミティブ型の例
int userAge = 25;
boolean isActive = true;

// 参照型の例
String userName = "Alice";
```

## 2. 変数と定数の宣言

Javaでは変数を使用する前に、必ずその型を宣言しなければなりません。

```java
// 変数宣言
int score = 80;
score = 90; // 上書き可能

// 定数宣言（再代入不可、JSのconstに相当）
// final 修飾子を付けます
final double PI = 3.14159;
// PI = 3.14; // <- コンパイルエラーになる
```

## 3. 型推論 (var)

比較的新しいJava（Java 10以降）では、右辺から明らかに型が分かる場合に限り、TypeScriptのように `var` を使った「型推論」が許されています。
（※JSの古い `var` とは性質が異なり、再代入の安全性などは変わりません。書くのが楽になるだけです）

```java
// String型であることが推論される
var greeting = "Hello, Java!";
// greeting = 100; // <- 元々Stringなので、これはコンパイルエラーになる（厳格）
```
