# 03. 次世代AIプロダクトの Generative UI/UX

AIプロダクトのフロントエンド（Next.js / React 等）におけるUI/UXパラダイムシフトについて解説します。
従来の「ボタンを押して、APIからJSONが返ってくるのを待ち、画面を描画する」というクラシックなWebの常識は、レスポンスに多大な時間がかかるLLMの前では崩壊しました。

## 1. Streaming UI (ストリーミングUI) と SSE

AI（LLM）は全文を一気に生成するのではなく、最初の1単語目を思いついた瞬間にそれを吐き出し、次の単語を数珠つなぎに生成し続けます。これを**トークン**と呼びます。

### 概念

フロントエンドでは、APIのレスポンス完了（200 OK）を待つのはアンチパターンです。**SSE (Server-Sent Events)** などの技術を使い、サーバーから1文字ずつポロポロと降ってくるテキストチャンクをReactの `useState` などでリアルタイムに画面に結合・描画し続ける「ストリーミングによる体感速度の向上」が必須となります。
Next.jsのApp Router (`AI SDK` など) は、このストリーミングを極限までシンプルに扱えるように作られています。

## 2. Generative UI (UIの動的生成)

Vercelの `v0` のようなサービスが代表的ですが、AIが返すのは「テキストの文章」だけではありません。

### パラダイムシフト

React（Next.js RSC）の普及により、**AIが「Reactコンポーネントそのもの」をリアルタイムに生成（あるいはバックエンドで組み立てて）クライアントにストリーミング配信する**ことが可能になりました。
ユーザーが「私の今月の売上をグラフで見せて」とチャットに打ち込むと、AIは「売上は100万円です」と文字で答えるだけでなく、実際にマウスでホバーできるインタラクティブな `Chart.js` のグラフコンポーネントそのものをチャット欄に出現させます。

これが **Generative UI** です。LLMは回答と同時にコンポーネントのステートやpropsを生成し、フロントエンドはそれを `Suspense` や `RSC Payload` として即座にレンダリングします。

## 3. Optimistic UI (楽観的UI)

LLMアプリの最大のUXの敵は「人間が待たされること」です。

### 魔法の演出

ユーザーが「この記事を保存して」とボタンを押した際、裏側でAIが評価処理等を行っていて数秒かかる場合でも、フロントエンドのReact側では**「APIの成功を待たずに、押された瞬間に『保存済み』のアイコンに切り替える」**という「楽観的な（成功すると信じて疑わない）」画面更新を行います。
もし裏側のAPIがエラー（失敗）を返してきた場合のみ、こっそりと「保存できませんでした」と元のアイコンに戻します。

これはReactの `useOptimistic` フック等で標準サポートされており、待ち合わせの多いAIプロダクトにおいてユーザーの離脱を防ぐための極めて重要なフロントエンド・テクニックです。
