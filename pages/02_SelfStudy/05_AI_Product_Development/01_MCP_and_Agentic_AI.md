# 01. MCP (Model Context Protocol) とエージェントAI基盤

このページでは、2026年時点の最新AIアーキテクチャである「MCP」と「Agentic AI（自律型AI）」について、インフラエンジニアやWeb開発者の視点からその本質を徹底解説します。

## 1. 歴史と誕生背景 (なぜMCPは生まれたのか？)

2023年頃から、AIに自社独自の機能（社内DBの検索や、Slackの投稿など）を持たせるために「**Function Calling (Tools Calling)**」という技術がもてはやされました。
これは、ユーザーが「明日の天気を教えて」と聞いた際に、AIが「天気を調べるAPIを叩いてくれ」とシステム側に要求し、システムがAPIを叩き、その結果をAIに返して回答させるという複雑な往復処理でした。

しかし、このアプローチには致命的な問題がありました。
「すべての開発者が、自分のアプリにAIを組み込むたびに、その独自のAPIの型定義やプロンプト指示をゼロから一生懸命書かなければならない」という再発明の嵐が起きていたのです。

> 📖 **補足**: MCPが標準化した現在でも、Function CallingはSDK直接利用型のCLIツール等では現役の主力手法です。MCP非対応の環境やシンプルなCLIエージェントでは、Function Callingの直接実装が最も効率的なケースが多くあります。
> 実装パターンについては **[Function Calling (Tool Use) 実装パターン](../09_CLI_Agent_Development/01_Function_Calling_Implementation)** をご参照ください。

そこで2024年末、Claudeを開発する **Anthropic** 社が中心となって提唱・オープンソース化した標準規格が **MCP (Model Context Protocol)** です。
MCPは「AI」と「外部のデータソース（ローカルPC、DB、GitHub、Slack等）」を繋ぐための「世界共通のコンセント（USBポートのようなもの）」として誕生しました。

## 2. 概念と動作原理

MCPは、非常にシンプルに言えば「Client-Serverモデル」です。HTTPではなく `stdio` (標準入出力) や SSEを利用してセキュアにローカル接続を行います。

### MCP Server (ツール提供者)

「自分のPCの中のファイルを読み書きする機能」「SQLiteの中身を検索する機能」などを、MCPの規格に則ったJSONスキーマ形式のインターフェースとして公開する小さなプログラムです。TypeScriptやPythonで簡単に作れます。

### MCP Client (AIエージェント)

Claude DesktopアプリやCursor、CLIツールなどのAI側です。起動時に指定された「MCP Server」に接続し、**「どんなツール（関数）が使えるのか」「どんなリソース（ファイル）が読めるのか」を自動で把握**します。

### 動作フロー

1. AIがユーザーから「プロジェクトのREADMEを要約して」と頼まれる。
2. AIは、自分が接続している `file-system-mcp-server` が提供している `read_file` ツールを自律的に呼び出す。
3. MCPサーバーがファイルを読み取り、AIにテキストを渡す。
4. AIが要約してユーザーに回答する。

## 3. 次世代パラダイム: Agentic AI (自律型エージェント)

従来の「チャットボット」は、RAGなどを駆使して「あらかじめ必要な情報を全て集めてからプロンプトに突っ込む」必要がありました。
Agentic AI（エージェント型のAI）は違います。

- **自律的思考と反復 (ReAct等)**
  「ゴール（要件）」だけを与えられます。AIは自ら足りない情報を探し（MCPツールで検索）、エラーが出たらそのエラーを読み込んでコードを書き直し（MCPツールでファイル編集）、テストを実行し（MCPツールでコマンド実行）、完了するまで自問自答をループします。
- **DevToolsの統合**
  現代の開発環境においては、人間がコードの一部を書いてAIに修正してもらうのではなく、MCP経由で「権限を与えられたAI」がエンジニアの一員としてターミナルを叩き、Dockerを立ち上げ、プルリクエストを作成するところまでを自律的に行うのが最新のAIプロダクトの形です。

## 4. 強みと弱み (Pros/Cons)

| 特徴 | 強み (Pros) | 弱み (Cons) |
| :--- | :--- | :--- |
| **標準化 (MCP)** | 一度 `Slack-MCP-Server` を作れば、ClaudeでもCursorでもどのAIクライアントからでも使い回せる。圧倒的な開発エコシステムの統一。 | 非常に低レイヤーの権限操作をAIに委譲するため、セキュリティ上のサンドボックス設計が極めて難しい（AIが勝手にシステムファイルを消す等）。 |
| **Agentic AI** | 人間の指示が「超アバウト（ふわっとした要件）」でも、勝手に解釈して複雑な複数ステップのタスクを完遂してくれる。 | 一歩間違うと無限ループに陥り、APIのトークン課金を数時間で何万円も消費してしまう暴走リスクがある（いわゆる "Token Limits" による安全装置が必須）。 |
