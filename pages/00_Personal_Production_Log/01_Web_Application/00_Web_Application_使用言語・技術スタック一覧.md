# Web_Application (使用言語・技術スタック一覧)

本ディレクトリは、自己学習およびAIアシスタントとの協業によって構築したWebアプリケーションのプロダクションログを管理する。

---

## 1. 開発プロジェクトと採用技術

### プロジェクトA: The Vocaloid Times

Next.js (App Router) を基盤とした、記事閲覧および持続的音声再生 (PWAやSPAの特長を生かしたセッション維持) 機能を持つ最新鋭のWebメディア構築テスト。

- **フロントエンド**: Next.js 16 (React 19), TypeScript
- **スタイリング**: Tailwind CSS 4
- **バックエンド/BaaS**: Supabase (データ管理・認証)

### プロジェクトB: May be ごいた Online

能登地方の伝統ゲーム「ごいた」のロジックを解釈し、オンラインでのリアルタイム同期対戦機能を持つWebアプリケーションとして完全構築。

- **フロントエンド**: Vanilla JavaScript, HTML, CSS (Viteビルド)
- **バックエンド/通信**: Firebase Realtime Database
- **特記事項**: AI (Antigravity) への自然言語でのルール説明と要件定義のみから構築された完全生成事例。

## 2. 実務に通じる技術選定の前提思考

上記の個人プロジェクトにおいて採用された技術スタックは、現代の実務現場においても極めて需要が高く、合理的である。

- **「TypeScript + React/Next.js」の選択**: 大規模開発における型安全性の確保と、コンポーネント指向による保守性の向上という、モダンフロントエンドの「最適解」をそのまま採用している。
- **「Supabase / Firebase」の選択**: アプリケーションの仮説検証 (MVP開発) フェーズにおいて、バックエンドの環境構築やインフラ保守に時間をかけず、フロントエンドのコア機能開発にリソースを全振りするBaaS (Backend as a Service) の利用は、スタートアップ企業などの実務において必須の戦略である。
