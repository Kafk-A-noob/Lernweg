# Infrastructure (前提知識と構成設計)

本ディレクトリは、自己学習および実証実験として構築した、物理サーバー (オンプレミス環境) からのインフラストラクチャ構築と運用設計のログを管理する。

---

## 1. オンプレミス環境構築の前提思想

Webアプリケーション開発においてクラウド (AWSやGCP、Vercel等) の利用が当たり前となった現代においても、その基盤であるLinux OSのネットワーク設定やポート開放・トンネリングの仕組みといった低レイヤーのインフラ知識は、複雑なトラブルシューティングにおいて必須となる。

本ログでは、市販のPCパーツと100円均一のワイヤーネットを活用した自作の完全ヘッドレス (モニタ・キーボード無し) 環境に、Ubuntu Serverをゼロからインストールし、セキュリティを担保した上で外部公開するまでの泥臭いインフラ構築の全体像を記録している。

---

## 2. 採用されたインフラ技術スタック

本環境 (Minecraft Modサーバー運用) において採用された技術と、その選定理由は以下の通りである。

- **Host OS**: Ubuntu Server 24.04 LTS (CLI環境)
  - Desktop版を選ばず完全なCUI環境を選択することで、OSが消費するメモリを最小限 (2GB程度) に抑え、リソースをすべてアプリケーションに回す。
- **Container**: Docker / Docker Compose
  - Javaのバージョン依存関係や、サーバーファイルのディレクトリ汚染を防ぐため、アプリケーション環境をコンテナ技術によって完全に隔離 (カプセル化) する。これにより、不要になればコンテナを破棄するだけで環境がクリーンになる。
- **Security & Remote**: OpenSSH (公開鍵認証) / UFW
  - ハッカーからの辞書攻撃 (総当たり攻撃) を無効化するため、パスワードログインをシステムレベルで禁止 (`PasswordAuthentication no`) し、特定の秘密鍵を持つメインPC (Windows) からのみSSH接続を許可する堅牢な要塞化を実施。
- **Network Tunneling**: Playit.gg
  - 自宅のルーターに穴を開ける (物理的なポート開放を行う) ことによるネットワーク全体へのセキュリティリスクを回避するため、リバースプロキシの仕組みを持つトンネリングサービスを採用。安全に外部のプレイヤーを内部のコンテナに誘導する。
