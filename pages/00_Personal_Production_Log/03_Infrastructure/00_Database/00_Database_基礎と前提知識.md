# データベース 基礎と前提知識 (00_Database)

本ドキュメントは、システムが扱う「データ」を永続的かつ構造的に保存するための基盤技術、リレーショナル・データベース(RDB)およびSQL言語についての学習総括である。

---

## 1. データベースの役割とRDB

データベースとは、大量の情報を整理して巨大な倉庫に保管し、瞬時に検索・引き出しができるようにしたシステムのことである。

中でも現在最も標準的に使われているのが「リレーショナル・データベース (RDB)」である。
RDBは、エクセルやスプレッドシートのような「行 (レコード) 」と「列 (カラム) 」で構成された「表 (テーブル) 」の形式でデータを管理する。

### 主キー (Primary Key) の重要性

RDBにおける最も重要な概念が「主キー」である。
テーブルの中で「この1行を絶対に特定できる100%ユニークな印」のことを指す。 (例：社員番号、ユーザーIDなど)
【実務Tip】 主キーが存在しないテーブルは、データを更新・削除しようとした時に「同姓同名の別人のデータまで消してしまう」といった致命的な事故を引き起こすため、実務ではいかなるテーブルにも必ず一意の主キー (通常は連番のID) を持たせることが絶対のルールである。

---

## 2. SQL (Structured Query Language) の基本操作 (CRUD)

データベースを操作するための専用言語がSQLである。
Webシステムにおいてデータベースに行う操作は、突き詰めれば以下の4種類 (CRUD) に集約される。

### ① CREATE / INSERT (作成・追加)

テーブルに新しい行 (データ) を追加する。

```sql
INSERT INTO persons (name, age) VALUES ('Yamada', 25);
```

### ② READ / SELECT (読み取り・検索)

テーブルから条件に合うデータを抽出する。実務で最も書く機会が多いSQL文である。

```sql
-- personsテーブルから、全カラムのデータを取得する
SELECT * FROM persons;

-- 条件付き検索 (20歳以上のみ)
SELECT name, age FROM persons WHERE age >= 20;
```

### ③ UPDATE (更新)

既存のデータを書き換える。

```sql
-- IDが1の人物の年齢を30に変える
UPDATE persons SET age = 30 WHERE id = 1;
```

【実務Tip】 `UPDATE` 文を書く際、末尾の `WHERE` (条件指定) を書き忘れると、テーブルにいる全員の年齢が30歳に書き換わってしまう (全件更新事故)。そのため、更新時は必ず「主キー」をWHEREで指定して1件だけを狙い撃ちにするのが鉄則である。

### ④ DELETE (削除)

既存のデータを削除する。

```sql
-- IDが2の人物を削除する
DELETE FROM persons WHERE id = 2;
```

【実務Tip】 これもUPDATE同様、WHEREを忘れると全データが消滅する。
現代の実務システムでは、データ復旧ができなくなるのを防ぐため、本当に DELETE を実行してデータを消滅させることは少ない (物理削除)。代わりに「is_deleted」といったON/OFFのカラムを用意し、UPDATE句でそこをTrueにするだけの「論理削除」を採用するケースが非常に多い。

## 学習のまとめ

近年のWeb開発 (特にFlaskやNext.jsなどのフレームワーク) においては、SQLを直接書かずにORM (オブジェクト関係マッピング) で操作する手法が主流になりつつある。(Flask編 02\_データベースとORM 参照)
しかし、ORMが裏側で最終的に組み立てているのはこのSQL文そのものである。バグを調査したり、複雑な検索処理のパフォーマンスを改善する際には、基礎となる生SQLの知識が必ず要求される。
